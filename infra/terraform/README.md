# TeamFit POC Terraform

Creates low-cost POC topology:

- Static Web App frontend
- Linux App Service B1 API with managed identity and VNet Integration
- Azure SQL Database, Cosmos DB, Storage Account, Key Vault via Private Endpoint + Private DNS
- Azure AI Services / Azure OpenAI via Private Endpoint + Private DNS
- Log Analytics + Application Insights

Default resource group: `rg-verde`.

Remote state backend matches `.github/workflows/terraform-poc.yml`:

| Setting | Value |
|---|---|
| Resource group | `rg-verde` |
| Storage account | `tfstateverde` |
| Container | `tfstate` |
| Key | `teamfit-poc.tfstate` |

## Commands

```pwsh
terraform init
terraform fmt -recursive
terraform validate
terraform plan -out=plan.out
```

For local use, backend storage must already exist. The GitHub Actions workflow bootstraps it automatically before `terraform init`.

No `terraform apply` for MVP unless explicitly approved.

## Static Web App API routing

Default keeps Static Web App on Free tier:

```hcl
static_web_app_sku_tier = "Free"
static_web_app_sku_size = "Free"
enable_static_web_app_backend_link = false
```

For deployed Free-tier frontend, set `frontend_api_base_url` to `https://<api-app>.azurewebsites.net/api` and include the Static Web App hostname in `allowed_cors_origins` in a follow-up plan, or inject equivalent values in CI/CD. This avoids a Terraform dependency cycle because both hostnames are generated by Azure.

Azure Static Web App linked backend requires Standard. Enable when `/api/*` proxying is needed:

```hcl
static_web_app_sku_tier            = "Standard"
static_web_app_sku_size            = "Standard"
enable_static_web_app_backend_link = true
```

## Key Vault secrets

Default does not write secrets from Terraform:

```hcl
manage_key_vault_secrets = false
```

Reason: Key Vault public access is disabled and secret writes use the data plane. If `manage_key_vault_secrets = true`, run Terraform from inside the VNet/private DNS path or use a controlled bootstrap.

## SQL password

If `sql_admin_password` is null, Terraform generates one. Generated passwords remain in Terraform state; use secure remote state before real apply.

## Security warning

The API stays public for low-cost POC. Current MVP auth is mock `X-User-Id`, so applied environments are demo-only until real auth or private ingress is added.

## Azure auth

Use Azure CLI or environment variables:

```pwsh
az login
az account set --subscription <subscription-id>
terraform plan -var="subscription_id=<subscription-id>"
```
